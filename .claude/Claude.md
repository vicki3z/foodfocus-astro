## Project Overview

SSG website for Food Focus Thailand built with Astro 5 + React 19 + TypeScript. Headless WordPress backend (`foodfocusthailand.com/wp-cms`) with data fetched via GraphQL at build time. Deployed to an FTP server (no server-side runtime).

## Commands

| Command | Purpose |
|---|---|
| `pnpm dev` | Start dev server |
| `pnpm build` | Production build (static output to `dist/`) |
| `pnpm preview` | Preview production build locally |
| `pnpm codegen` | Regenerate GraphQL types into `src/types/generated/graphql.ts` |

**Note:** Node 20, pnpm 10. No test runner is configured.

## Architecture

### Data Flow

1. **GraphQL queries** defined in `src/lib/queries.ts` as `gql` tagged templates
2. **Client & fetchers** in `src/lib/graphql.ts` — two main functions:
   - `fetchGraphQL<T>()` — single query execution
   - `fetchAllPaginated<T>()` — cursor-based pagination (handles `hasNextPage`/`endCursor`)
3. **Astro pages** call these in `getStaticPaths()` frontmatter, map responses to props, pass to components
4. **Types** generated by `@graphql-codegen/cli` from the WordPress schema → `src/types/generated/graphql.ts`

After changing GraphQL queries, run `pnpm codegen` to regenerate types.

### Component Model

- **Astro components** (`.astro`) — static/structural: layouts, grids, cards, pagination
- **React components** (`.tsx`) — interactive: header/navigation, carousels, tabs, search, year selectors
- Client directives: `client:load` for immediately interactive (Header, YearSelector), `client:visible` for lazy-loaded (BannerCarousel)

### Rendering Strategy

All pages are statically generated (`output: "static"`). Pagination is computed at build time — pages like `news/page/[page].astro` chunk fetched posts and generate one HTML file per page. Index pages (e.g. `news/index.astro`) redirect to `page/1` via meta refresh.

### Environment Variables

- `WP_GRAPHQL_URL` — WordPress GraphQL endpoint (required for build and codegen)
- `PUBLIC_GOOGLE_MAPS_API_KEY` — Google Maps (prefixed `PUBLIC_` for client access)

### Deployment

GitHub Actions (`deploy-ftp.yml`) deploys via FTPS in three parallel jobs splitting root, `news/*`, and `whats-in/*` directories. Triggered by: push to main, hourly cron (7am–7pm Bangkok time), manual dispatch, WordPress `repository_dispatch` webhook.

## Key Patterns

- `enforceHttps()` in `src/lib/graphql.ts` rewrites WordPress URLs from HTTP to HTTPS — always use this when processing WordPress media URLs
- WordPress custom post types (roadmaps, roadshows, proseries, seminars, exhibitions) each have dedicated queries and page routes
- Radix UI primitives (`NavigationMenu`, `Tabs`, `Slot`) are used unstyled with Tailwind — don't add component library CSS
- Tailwind CSS v4 is configured via `@tailwindcss/vite` plugin (not PostCSS)
